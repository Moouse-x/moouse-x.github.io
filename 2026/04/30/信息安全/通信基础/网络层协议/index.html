IP协议、ICMP协议、ARP协议

网络层的协议

IP协议

总长度：表示IP报文的所有内容所组成的报文长度,20-60字节之间，40为选项的长度
IP报文的格式
固定长度：总长度除去IP Opthins 选项（可能是空）
version版本号
一般都是4–ipv4
Header Length头部长度
固定是20Bytes
DS Field报文的类型
IP分片
Identification标识符
区别于同一主机或其他主机发送的其它数据包分片
Flags标记
0代表传输完毕；1代表数据还在传输过程中
Fragment Offset片偏移
表示每个分片在原始报文中的位置
Time to Live生存时间
当报文的传输长度超过了TTL值，这个报文就会丢包（TTL=0）
Protocol上层使用的协议
TCP(0x06)和UDP(0x11)
Header Checksum
Source IP Address源地址
Destination IP Address目的地址

ICMP协议

互联网控制消息协议 (ICMP) 是一组通信规则，设备使用这些规则在网络中传达数据传输错误。在发送方和接收方之间的消息交换中，可能会出现某些意外错误。
传递差错、控制、查询等信息。
ping
ICMP Echo请求消息(Type值为8)
ICMP Echo回复消息(Type值为0)
tracert
追踪网络数据包从源地址到目的地址所经过的路径（路由）
ICMP重定向(Redirect)
路由器通知主机存在更有的路由路径，减少不必要的转发
ICMP差错检测
ping
ICMP Echo Request和ICMP Echo Reply消息常用于诊断源和目的地之间的网络连通性
ICMP数据包格式
Type
表示ICMP消息类型
回显请求(Echo Request)和回显应答(Echo Reply)
目的地不可达(Destination Unreachable)
超时(Time Exceeded)
重定向(Redirect)
Code
表示同一消息类型的不同信息
Checksum
校验和：用于验证消息的完整性
Message Body
消息体：包含具体的控制信息或错误数据。

ARP协议

Address Resolution Protocol
是一种用于将网络层地址（如 IP 地址）解析为数据链路层地址（如 MAC 地址）的协议
通过目的端的IP地址来获取目的端的MAC地址
