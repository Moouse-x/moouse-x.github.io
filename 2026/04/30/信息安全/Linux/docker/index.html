一、简介

理解

docker是轻量级的虚拟化基础，让应用程序及其依赖环境可以打包成一个标准化的单元,在任何地方都能一致地运行
docker->软件
集装箱->货物

二、基本概念

1.概念

镜像（image）
Docker 镜像是一个特殊的文件系统，除了提供容器运行时所需的程序、库、资源、配置等文件外，还包含了一些为运行时准备的一些配置参数 (如匿名卷、环境变量、用户等)。
像活字模板
镜像就是已经刻好字的字模
是只读的不能更改，包含了所有信息（油墨颜色、字体大小、内容）
容器（container）
镜像 (Image) 和容器 (Container) 的关系，就像是面向对象程序设计中的类和实例 一样，镜像是静态的定义，容器是镜像运行时的实体。容器可以被创建、启动、停止、删除、暂停等。
像印出来的字帖
动态的可以拿笔在纸上涂改（在容器里写日志、传文件）但这些修改不会影响到字模本身，可以打印100份就相当于有100个容器，每一份都是独立互不干扰的
仓库（repository）
镜像构建完成后，可以很容易的在当前宿主机上运行，但是，如果需要在其它服务器上使用这个镜像，我们就需要一个集中的存储、分发镜像的服务，Docker Registry 就是这样的服务。
存放字模的图书馆、仓库
需要某个环境时直接去找一个现成的。

3.容器

本质是一个特殊的进程
进程空间-容器看不到宿主机上的其它进程
网络-容器拥有独立的IP 、端口等网络资源
文件系统-容器拥有独立的root目录
用户-容器内的root用户不等于宿主机的root用户
容器的存储层
镜像层+容器层
docker会在镜像的只读层之上创建一个可写层（容器存储层）
copy-on-write：写时复制
容器需要修改镜像层中的文件
1.docker将文件复制到容器存储层
2.在容器层中进行修改
3.原始镜像层保持不变
容器存储层的生命周期
容器存储层和容器生命周期绑定。容器删除的时候数据也删除了
所有持久化的数据需要
数据卷 docker管理的存储 数据库、应用程序
绑定挂载 挂载宿主机目录 开发时共享代码
docker run -v mydata:/var/lib/mysql mysql
docker run -v /host/path:/container/path nginx

4.仓库

registry
存储和分发docker镜像的服务
镜像构建完成后，可以在当前机器上运行；但要在其他服务器上运行就需要一个集中的存储和分发服务–Docker Registry
registry 仓库 标签
Docker Registry中可以包含多个Repository，每个Repository可以包含多个Tag
镜像的完整名称
[registry 地址/][用户名/]仓库名[:标签]
registry.example.com/mycompany/myapp:v1.2.3
镜像的推送和拉取
docker login--登录Docker Hub docker login registry.example.com–登录其他registry docker pull nginx:1.25–拉取镜像 docker tag myapp:latest registry.example.com/myteam/myapp:v1.0–标记镜像（准备推送） docker push registry.example.com/myteam/myapp:v1.0–推送镜像 docker logout–登出漏洞扫描 docker scout cves nvinx:latest--docker scout扫描镜像漏洞 trivy image nginx:latest`–开源工具trivy

三、安装Docker

Linux

Ubuntu

1.先决条件
sudo apt install gnome-terminal
2.安装Docker Desktop
# Add Docker's official GPG key: sudo apt update
sudo apt install ca-certificates curl sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc sudo chmod a+r /etc/apt/keyrings/docker.asc
# Add the repository to Apt sources: sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}") Components: stable
Architectures: $(dpkg --print-architecture) Signed-By: /etc/apt/keyrings/docker.asc
EOF sudo apt update 安装docker软件包 sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin 查看docker是否运行 sudo systemctl status docker ==注意== 国内环境无法直接访问Docker Hub官方镜像库，需要配置国内镜像加速器 sudo mkdir -p /etc/docker
sudo nano /etc/docker/daemon.json
{ `` "registry-mirrors": [ “https://docker.m.daocloud.io“,
"https://huecker.io", “https://dockerhub.timless.top“,
"https://noohub.net" ] }
重启docker服务使配置生效
sudo systemctl daemon-reload sudo systemctl restart docker
3.卸载docker engine
sudo apt purge docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin docker-ce-rootless-extras
删除所有的镜像、容器和卷
sudo rm -rf /var/lib/docker sudo rm -rf /var/lib/containerd 删除源列表和密钥环 sudo rm /etc/apt/sources.list.d/docker.sources
sudo rm /etc/apt/keyrings/docker.asc

四、使用镜像

1.获取镜像

docker pull 命令
docker pull [选项] [Registry地址/]仓库名[:标签]
docker pull --all-tags,-a–拉取所有镜像
docker pull --platform–指定平台架构
docker pull --quiet,-q–静默模式
镜像名称格式
docker.io / library / ubuntu : 24.04
registry地址 用户名 仓库名 标签
前两个可省略
拉取后运行
docker run -it --rm ubuntu:24.04 bash
-it交互式终端模式
--rm退出后自动删除容器
bash启动命令
==docker run在需要时会自动pull镜像，因此通常不需要单独执行docker pull==

2.列出镜像

查看本地已下载的镜像
docker image ls/docker images
REPOSITORY 仓库名
TAG 标签（版本）
IMAGE ID 镜像唯一标识（短ID，前12位）
CREATED 创建时间
SIZE 本地占用空间
