请求消息

请求报文一般格式
![[HTTP请求消息格式.png]]
客户端发送一个HTTP请求消息包括
请求行 request line
请求头部 request header
空行
请求数据
请求行
方法 GET、POST、PUT、DELETE等，指定要执行的操作。
请求URI （统一资源标识符）：请求的资源路径，通常包括主机名、端口号、路径和查询字符串
HTTP版本：HTTP/1.1或HTTP/2
如GET /index.html HTTP/1.1
请求头
包含客户端环境信息、请求体大小、客户端支持的压缩类型
常见有Host User-Agent Accept Accept-Encoding Content-Length
空行
请求头和请求体之间的分隔符，表示请求头结束
请求体
在某些类型的HTTP请求种，请求体包含要发送给服务器的数据

响应消息

状态行(Status Line)
HTTP版本：与请求消息中的版本匹配
状态码：三位数，表示请求的处理结果。如200表示成功，404表示未找到资源
状态信息：状态码的简短描述
如HTTP/1.1 200 OK
响应头(Response Headers)
包含了服务器环境信息、响应体大小、服务器支持的压缩类型
如Content-Type Content-Length Server Set-Cookie
空行
响应头和响应体之间的分隔符，表示响应头的结束
响应体
包含服务器返回的数据，如请求的网页内容、图片、JSON数据等
实例
客户端请求
GET /index.html HTTP/1.1—请求行
请求方法 访问路径 版本
Host: www.example.com—请求头
目标主机
User-Agent: Mozilla/5.0 (Windows NT 10.0;Win64; x64;rv:91.0)Gecko/20100101 Firefox/91.0—请求头
浏览器标识Accept:text/html,application/xhtml+xml,application/xml;q=0.9,imag``e/webp,*/*;q=0.8—请求头
告诉服务器：我想要什么类型的数据
Accept-Encoding: gzip, deflate—请求头
压缩方式
Connection: keep-alive—请求头
是否保持连接
服务器响应
HTTP/1.1 200 OK—状态行
HTTP版本 状态码 状态描述
Date: Wed, 18 Apr 2024 12:00:00 GMT—响应头
Server: Apache/2.4.1 (Unix)—响应头
服务器信息

Content-Length: 12345—响应头
返回数据长度
Content-Type: text/html; charset=UTF-8–响应头
返回内容类型
—空行
